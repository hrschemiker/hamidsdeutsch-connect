# HamidsDeutsch Connect — چک‌لیست آزمون یکپارچه نهایی

> این مرحله فایل نصبی تولید نمی‌کند. هدف آن اطمینان از پایداری نسخه فعلی پیش از ورود به مرحله انتشار است.

## 1. آزمون خودکار Preflight

از ریشه پروژه اجرا شود:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\release-preflight.ps1
```

نتیجه قابل قبول:

- `Overall: READY`
- یا `Overall: READY WITH WARNINGS`، فقط وقتی هشدار مربوط به تغییرات ثبت‌نشده Git باشد.
- هیچ مورد `FAIL` نباید باقی بماند.

گزارش‌ها در پوشه زیر ذخیره می‌شوند:

```text
release-reports
```

## 2. شروع تمیز برنامه

1. Windows Proxy خاموش باشد.
2. هیچ `sing-box.exe` قدیمی در Task Manager نباشد.
3. برنامه با `npm start` اجرا شود.
4. اشتراک‌ها فقط یک‌بار بارگیری شوند.
5. حلقه تکراری `Safe nodes loaded` دیده نشود.
6. سرورها بر اساس پینگ مرتب شوند.

## 3. اتصال خودکار

1. حالت اتصال روی `خودکار` باشد.
2. برنامه با Administrator اجرا شود.
3. ابتدا TUN آزمایش شود.
4. اتصال تنها پس از تغییر واقعی IP موفق اعلام شود.
5. نام سرور، اشتراک، پینگ و حالت اتصال درست نمایش داده شوند.
6. صفحه آمار یک نشست جدید ثبت کند.
7. صفحه گزارش رویداد موفق را ثبت کند.

## 4. System Proxy fallback

1. برنامه بدون Administrator اجرا شود؛ یا TUN عمداً غیرقابل‌استفاده باشد.
2. حالت اتصال روی `خودکار` و fallback روشن باشد.
3. اتصال از System Proxy برقرار شود.
4. پس از قطع، تنظیمات Proxy قبلی ویندوز بازیابی شوند.
5. در تنظیمات Windows هیچ Proxy جا‌مانده‌ای وجود نداشته باشد.

## 5. حالت فقط TUN

1. حالت `فقط TUN` انتخاب شود.
2. بدون Administrator اتصال امتحان شود.
3. پیام واضح دسترسی Administrator نمایش داده شود.
4. System Proxy مخفیانه فعال نشود.
5. با Administrator اتصال TUN آزمایش شود.

## 6. حالت فقط System Proxy

1. حالت `فقط System Proxy` انتخاب شود.
2. اتصال برقرار شود.
3. TUN اصلاً راه‌اندازی نشود.
4. پس از قطع، Proxy ویندوز بازیابی شود.

## 7. مرکز نجات اتصال

1. مرکز نجات خاموش باشد و یک اتصال عادی آزمایش شود.
2. فقط `TLS Record Fragment` فعال شود.
3. اتصال قطع و دوباره برقرار شود.
4. سپس Fragment کامل فقط برای تست فعال شود.
5. SNI نامعتبر امتحان شود و خطای معتبر دریافت شود.
6. SNI خالی شود و تنظیمات بازنشانی شوند.

## 8. Split Tunneling

1. یک دامنه آزمایشی به سایت‌های مستقیم اضافه شود.
2. اتصال جدید برقرار شود.
3. دامنه مستقیم از مسیر VPN عبور نکند.
4. دامنه حذف و اتصال دوباره ساخته شود.
5. سایر سایت‌ها همچنان از تونل عبور کنند.

## 9. آخرین سرور موفق

1. به یک سرور مشخص با موفقیت وصل شوید.
2. اتصال قطع شود.
3. برنامه بسته و دوباره اجرا شود.
4. کارت سرور قبلی همان سرور را نمایش دهد.
5. دکمه اتصال به سرور قبلی کار کند.

## 10. Auto-Reconnect

1. اتصال واقعی برقرار شود.
2. اینترنت یا سرور برای مدتی قطع شود.
3. برنامه خرابی اتصال را تشخیص دهد.
4. نشست قبلی در گزارش بسته شود.
5. بازیابی خودکار شروع شود.
6. اتصال جعلی نمایش داده نشود.

## 11. Crash Recovery

1. اتصال واقعی برقرار شود.
2. Electron از Task Manager به‌صورت اجباری بسته شود.
3. برنامه دوباره اجرا شود.
4. sing-box جا‌مانده شناسایی و بسته شود.
5. Windows Proxy بازیابی شود.
6. فایل مالکیت Runtime پس از بازیابی پاک شود.

## 12. افزونه مکان مجازی مرورگر

1. افزونه فقط یک‌بار با `Load unpacked` نصب شود.
2. HamidsDeutsch قطع باشد.
3. افزونه وضعیت غیرفعال نشان دهد.
4. HamidsDeutsch متصل شود.
5. افزونه پس از بررسی دوباره فعال شود.
6. موقعیت HTML5 با کشور یا شهر IP خروجی هماهنگ باشد.
7. HamidsDeutsch قطع شود.
8. افزونه خودکار غیرفعال شود.
9. هنگام استفاده از VPN دیگر و قطع HamidsDeutsch، افزونه مکان را دست‌کاری نکند.

## 13. امنیت گزارش‌ها

گزارش فنی کپی شود و بررسی شود که موارد زیر در آن نباشند:

- Subscription URL
- VMess/VLESS/Trojan/Shadowsocks URI
- UUID
- Password
- Private key
- Token
- محتوای کامل کانفیگ sing-box

## 14. رابط کاربری

1. صفحه خانه عنوان و توضیح Hero حذف‌شده را نمایش ندهد.
2. دکمه اتصال اصلی واضح و قابل‌استفاده باشد.
3. لیست سرورها فشرده و مرتب باشد.
4. اطلاعات فنی فقط با Expand نمایش داده شوند.
5. صفحات خانه، سرورها، اشتراک‌ها، آمار، گزارش، نجات و تنظیمات بدون خطای ظاهری باز شوند.
6. اسکرول افقی ناخواسته وجود نداشته باشد.
